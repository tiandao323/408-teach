# 混合式讲解风格

本文件定义 `408-teach` 的讲解口吻和栏目结构。目标是避免“把教材换句话说一遍”，改成真正带学生学会。

## 总原则

讲解深度由 `重要性判断` 的 S/A/B/C/D 决定：

- S/A：从零推导 + 老师上课式，讲透。
- B：解题导向速通，讲清它服务哪个高频点。
- C/D：保底识别，讲到能看懂题、能排除错项即可。

低频知识不删除，但不写成长篇深讲。

## S/A 级讲解栏目

S/A 级知识点的 `核心概念与深度讲解` 必须包含以下栏目：

```markdown
#### 你可能会卡在哪里

先指出初学者最容易困惑的点。

#### 从零推导

从“为什么需要这个概念/机制”开始，一步步推出定义和规则。

#### 机制拆解

拆清楚对象、边界、参与者、执行流程、条件和结果。

#### 例子跑通

用一个具体例子把推理链跑完，不能只举名词例子。

#### 408 怎么考

给选择题/综合题的判断口径、题目信号和排除错项方法。

#### 易错点

指出具体错法。
```

写法要求：

- 先说学生会误会什么，再给正确模型。
- 不要直接堆定义；定义必须从问题中长出来。
- 例子必须能走完过程，尤其是 Cache、页表、同步、TCP、图算法这类机制题。
- `408 怎么考` 必须写成可操作的判题规则。

## B 级讲解栏目

B 级知识点的 `核心概念与深度讲解` 必须包含：

```markdown
#### 一句话定位

一句话说清这个概念在本章中负责什么。

#### 和高频点的关系

说明它支撑哪个 S/A 知识点，或在哪类题里作为前置概念出现。

#### 看到题怎么处理

给识别信号和处理步骤。

#### 408 怎么考

只写最常见题型和判断口径。

#### 易错点

只写最可能错的一两点。
```

## C/D 级讲解栏目

C/D 级知识点的 `核心概念与深度讲解` 必须包含：

```markdown
#### 最低掌握

保底定义、出现条件、不要投入大量时间的理由。

#### 看到题怎么处理

告诉学生遇到这个名词时如何识别、如何排除明显错误选项。

#### 408 怎么考

如果低频但大纲列出，说明可能怎么以边角选择题出现。

#### 易错点

指出最容易混淆的相邻概念。
```

## 口吻

- 像老师在旁边讲，不像百科条目。
- 可以直接写“你容易把 A 和 B 混在一起，原因是……”
- 允许使用短句、类比、过程表和自绘图。
- 不使用空泛句：如“需要重点掌握”“注意理解”“可能会考”。
