# 408-teach 讲义模板草案

这个文件把当前 `408-teach` 的讲义格式单独抽出来，供你修改。

注意：目前 skill 真正执行时仍以 `SKILL.md` 的 `Lecture Contract`、`references/lecture-quality.md` 和 `scripts/validate_lecture.py` 为准。你改完本文件后，需要再同步改那几个地方，新的讲义格式才会正式生效。

## 当前硬性校验规则

- 全文推荐使用两个一级标题：第一个必须是 `# 408考试大纲`，第二个必须是 `# {{章节号}} {{章节标题}}`。
- 为兼容旧讲义，仍允许只有一个章节一级标题；但新讲义默认使用两个一级标题。
- 提取出来的每个教材标题都必须保留，顺序必须一致。
- 每个教材标题下必须先放该标题对应的教材原文，再放讲解。
- 每个教材标题后必须有一个 `重要性判断` 模块。
- 每个教材标题后必须有一个 `核心概念与深度讲解` 模块。
- 每个讲解模块必须包含 `408 怎么考`。
- 每个讲解模块必须包含以 `易错点` 开头的标题。
- 不允许留下 `TODO`、`TBD`、`待补充`、`待完善`、`{{占位符}}` 这类占位内容。
- Markdown 代码围栏必须成对闭合。

## 可自由调整的部分

- `核心概念与深度讲解` 这个模块下面的子栏目数量和顺序。
- 是否保留 `这段在说什么`、`为什么需要它`、`直观例子` 等小标题。
- 是否增加局部例题、对比表、口诀、流程图、真题提示等栏目。
- 每个栏目里的写作口吻、详略程度和排版风格。

## 目前不建议随便改的部分

- 新讲义允许两个一级标题：`# 408考试大纲` 和 `# {{章节号}} {{章节标题}}`。
- 第一个一级标题必须精确写作 `408考试大纲`；第二个一级标题必须包含目标章节号。
- `408考试大纲` 下必须有 `408os 考频分析`。
- 教材原文必须在讲解前。
- 每个教材标题都要有 `重要性判断`。
- 每个教材标题都要有就地讲解。
- `408 怎么考` 和 `易错点` 这两个标题目前是验证器硬检查项。
- 如果你想把 `核心概念与深度讲解`、`408 怎么考`、`易错点` 改名，需要同步修改验证器。
