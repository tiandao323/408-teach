# 408os 考频分层讲解规则

本文件定义 `408-teach` 如何使用 `https://www.408os.cn/analysis` 的真题统计数据来决定讲义讲解深度。

## 数据来源优先级

1. 官方 408 大纲和项目登记的本地大纲文件决定“是否属于考试范围”。
2. `408os.cn/analysis` 的 2009-2026 真题统计用于判断“投入多少讲解篇幅”。
3. 项目登记的真题 PDF 用于补充题型、设问方式和陷阱；不得虚构年份或题号。
4. 如果 408os 无法访问，必须在讲义中说明“未能读取 408os 考频数据”，然后用本地大纲和真题资料保守分层。

## 读取方式

访问 `https://www.408os.cn/analysis`，从页面脚本中定位 `assets/analysisData-*.js`。该数据包包含：

- `summary`：总题量、总分、年份范围、知识点总数；
- `top`：按分值排序的知识点，含题量、分值、考察年份数和年份列表；
- `freq.high / medium / low / rare`：高频、中频、低频、极少考察知识点；
- `chapter`：各科章节逐年分值。

讲义只需要摘取与当前教材节相关的知识点，不要把全站统计整段塞入讲义。

## 分层标准

| 等级 | 判断依据 | 讲义策略 |
|---|---|---|
| S 核心必考 | 考察不少于 10 年，或总分极高，或属于全站 TOP15 | 机制级深讲：从零直觉、精确定义、运作过程、图示/表格、例子、真题判断口径和易错陷阱都写全 |
| A 高频重点 | 考察 5-9 年，或近 5 年持续出现，或连接多个高频点 | 标准深讲：讲清原理、边界、典型例子和 408 常见设问 |
| B 支撑型中频 | 本身分值不高，但支撑 S/A 知识点 | 速通讲：说明它是什么、为什么为高频点服务、题目中如何识别 |
| C 低频边缘 | 考察 2-4 年，且不支撑核心机制 | 概念卡片：一句话定义、适用条件、易混对象，避免长推导和大例子 |
| D 极少/未考 | 仅考 1 年或从未考察 | 保底识别：知道名词、能排除明显错误选项即可，不展开深讲 |

## 每节讲义必须写的考频区块

在 `# 408考试大纲` 下加入：

```markdown
## 408os 考频分析

- 数据范围：2009-2026，共 18 年真题统计。
- 本节相关知识点：
  - `知识点名`：等级 S/A/B/C/D，题量 X，分值 Y，考察年份 Z 年。
- 本节讲解策略：哪些深讲，哪些速通，哪些只做保底识别。
```

如果当前小节无法和 408os 知识点精确对应，写：

```markdown
## 408os 考频分析

- 未找到与本节完全同名的 408os 知识点。
- 本节按大纲范围和教材位置暂定为 B/C/D，理由是……
```

## 每个教材标题下必须写的重要性判断

每个教材标题的原文之后、深度讲解之前，加入：

```markdown
### 重要性判断

- 等级：S/A/B/C/D
- 依据：408os 考频、408 大纲位置、真题/章节连接关系
- 学习策略：深讲 / 标准讲 / 速通 / 概念卡片 / 保底识别
```

## 不同等级的讲解深度

S 级必须包含：

- 从“为什么需要这个概念”讲起；
- 明确软件/硬件边界或数据结构/算法边界；
- 给出至少一个能跑通思维过程的例子；
- 给出 408 选择题或综合题的判断规则；
- 给出具体易错点，不写空泛提醒。

A 级必须包含：

- 清晰定义和边界；
- 一个简短例子或对比；
- 至少一种常见考法；
- 易混点。

B 级必须包含：

- 一句话定位；
- 它和 S/A 级知识点的关系；
- 题目中看到它时如何处理。

C 级只需要：

- 名词解释；
- 出现条件；
- 最容易混淆的一点。

D 级只需要：

- 保底识别句；
- 说明不建议投入大量时间；
- 如果大纲仍列出，则提醒“不能完全跳过”。

## 写作口吻

讲义要像老师带着学生学：

- 先说“你可能会卡在哪里”；
- 再拆“这个概念为什么出现”；
- 最后给“考试中怎么判断”。

避免只把教材换句话说一遍。
