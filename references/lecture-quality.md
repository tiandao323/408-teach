# 逐节精讲质量参考

本文件定义 `408-teach` 讲义的最低质量标准。讲义必须以完整教材原文为主线，在每个教材标题后立即加入对应的深度讲解。

## 目录

1. 结构契约
2. 原文与 OCR
3. 408os 考频分层
4. 深度讲解
5. 408 应试要求
6. 校验清单

## 结构契约

- 新讲义默认有两个一级标题：`# 408考试大纲` 和包含目标章节号的章节标题。
- 为兼容旧讲义，仍允许只有一个包含目标章节号的章节一级标题。
- 请求父节时，在同一讲义中包含其全部下级小节。
- 保留提取结果中的每个教材标题，并保持顺序。
- 每个教材标题下先放对应教材原文，再放 `核心概念与深度讲解`；不得把教材正文移到讲解之后。
- 每个教材标题下必须先给出 `重要性判断`，再进入 `核心概念与深度讲解`。
- 每个讲解模块必须包含 `408 怎么考` 和以 `易错点` 开头的子标题。
- 不使用旧式的统一学习目标、原文摘录、例题、自测和总结框架。

推荐结构：

```markdown
# 408考试大纲

本节对应的 408 大纲条目。

## 408os 考频分析

- 数据范围：2009-2026，共 18 年真题统计。
- 本节相关知识点：知识点名、等级、题量、分值、考察年份数。
- 本节讲解策略：深讲 / 标准讲 / 速通 / 概念卡片 / 保底识别。

# X.Y 教材标题

## 教材原文

按 408 大纲引用教材原文。

### 重要性判断
- 等级：S/A/B/C/D
- 依据：408os 考频、408 大纲位置、真题/章节连接关系
- 学习策略：深讲 / 标准讲 / 速通 / 概念卡片 / 保底识别

### 核心概念与深度讲解
#### 这段在说什么
通俗解释。
#### 为什么需要它
背景、作用或原理。
#### 直观例子
紧贴概念的例子。
#### 408 怎么考
题型、判断口径和常见设问。
#### 易错点
相邻概念、边界和陷阱。
```

内容简单时可合并可选子栏目，但通俗解释、`408 怎么考` 和 `易错点` 不得缺失。

## 408os 考频分层

讲义必须使用 `references/408os-importance.md` 的分层规则。

- S 核心必考：机制级深讲，必须从直觉、定义、运作过程、例子、考法和易错点完整展开。
- A 高频重点：标准深讲，讲清原理、边界、例子和常见设问。
- B 支撑型中频：速通讲，说明它支撑哪个高频点，以及题目中如何识别。
- C 低频边缘：概念卡片，只保留定义、出现条件和易混点。
- D 极少/未考：保底识别，不建议投入大量时间，但大纲列出时不能完全跳过。

低频知识点不得删除，但必须降级处理，避免挤占高频知识的讲解篇幅。

## 原文与 OCR

- 完整保留提取范围内的教材段落、公式、表格、代码和图片引用，不得用摘要替代原文；长节分块时必须先合并全部分块。
- 只修正有依据的 OCR 错误，优先处理章节号、术语、单位、上下标、符号和英文粘连。
- 可依据目录或相邻标题补回缺失标题，并简要说明依据；为保持只有 `408考试大纲` 和目标章节两个一级标题，可降级教材中的 Markdown 标题级别。
- 无法可靠恢复的截断必须标注，不得臆补。
- 概念性图示默认不要引用教材图片文件；按教材图意自己构造 Mermaid、表格、ASCII 图或简洁文字图，并保留有用的图号和图题。
- 只有当用户明确要求保留原图，或图片无法可靠重构为图示（如照片、复杂电路实物图、扫描手写批注、视觉密集表格）时，才保留教材中的本地图片链接；`finalize` 负责复制和改写为可移植链接。

## 深度讲解

每个模块先看重要性等级，再选择讲解深度。S/A 级至少覆盖以下角度：

- 用一句话说明原文解决什么问题。
- 先建立直觉，再给准确术语和定义。
- 解释为什么需要该概念、机制怎样运作、适用边界是什么。
- 用简短例子连接抽象概念与实际执行过程。
- 对比容易混淆的相邻概念。
- 有公式或算法时给出推导、变量含义、单位和适用条件；没有实质推导时不要硬补。
- 例题放在对应概念附近，并给出答案与理由，不虚构教材来源。

避免空泛复述原文。讲解必须增加理解所需的因果关系、判断方法或边界信息。B/C/D 级可以短，但不能只写“了解即可”；必须告诉学生看到题时如何处理。

## 408 应试要求

`408 怎么考` 至少给出以下一种有效内容：

- 选择题或判断题的判定口径；
- 常见设问形式；
- 计算题的步骤或公式入口；
- 综合题中与相邻章节的连接方式。

`易错点` 应指出具体错误，而不是只写“注意概念”。优先覆盖绝对化表述、条件遗漏、层次混淆、单位错误、对象或观察者不明确等常见陷阱。

## 校验清单

- 是否完整覆盖目标父节及其全部后代内容？
- 教材标题是否全部存在且顺序一致？
- 每个教材标题下的正文是否仍在对应讲解之前？
- 是否写了 `408os 考频分析`？
- 每个教材标题是否写了 `重要性判断`？
- 每个标题后是否紧邻非空的深度讲解？
- 每个讲解是否包含 `408 怎么考` 和 `易错点`？
- OCR 修正是否有依据，截断是否明确标注？
- 公式、术语、单位、例子和答案是否一致？
- 概念图是否已自绘而不是直接引用教材图片？若保留图片链接，是否确有必要且源文件存在？
- 是否仍有占位文字或未闭合代码围栏？
- 是否能通过 `scripts/validate_lecture.py` 和 `main.py finalize`？
